# 트레이아(Treia) 실행 계획서 (Implementation Plan)

## 1. 현재 상황 분석 (Strategic Pivot)

- **현상**: 대시보드 UI 및 기본 API 인프라 구축 완료.
- **핵심 목표**: **"보안 중심의 독자적 시그널 분산 및 카피트레이딩 시스템"** 구축.
- **방향**: 개인의 12년 경력 전략(EA) 소스코드 노출 없이, 다수의 유료 회원에게 실시간 매매 신호를 전달하고 자동 체결하는 플랫폼으로 확장.

## 2. 향후 릴리즈 계획 (2026.03.05 ~ 03.10)

### Phase 1: 시그널 허브(Signal Hub) 인프라 구축 (핵심)

- **마스터 EA (Signal Generator)**: 매매 발생 시 `treia` API로 데이터를 쏘는 통신 프로토콜 설계.
- **서버 API (Next.js/Firestore)**: 멀티 EA(골드, 나스닥 등) 대응 가능한 신호 수집 및 관리 로직 개발.
- **수신 EA (Receiver Skeleton)**: 소스코드가 없는 단순 실행용 EA 개발 및 배포 준비.

### Phase 2: 대시보드 미니멀라이즈 & 실시간성 확보

- **UI 간소화**: 트레이딩에 직결되는 [메인 차트 + 시그널 + 경제 지표] 외 불필요 요소 전면 삭제.
- **실시간성**: 텔레그램 웹훅 Or 마스터 EA 직접 통신으로 0.5초 내외의 신호 전달 속도 확보.

### Phase 3: 차트 매물대 박스 및 자동 분석 고도화

- **H1 매물대 연동**: TradingView 차트 위에 데이트레이딩용 박스 영역 자동 생성 및 시각적 가이드 강화.
