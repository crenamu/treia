# 핀테이블(FinTable) 실행 계획서 (Implementation Plan)

_최종 업데이트: 2026-03-13 (금) 17:55 KST_

---

## 1. 현재 상황 분석 (Pivot: Treia → FinTable)

- **배경**: 골드 EA 전문 서비스(Treia)에서 종합 금융 정보 큐레이션 플랫폼(FinTable)으로 확장.
- **핵심 목표**: "공공데이터(금감원, LH/SH) 기반의 투명한 금융 정보 제공 및 Treia의 서브 메뉴 내재화."
- **브랜딩**: 옅은 베이지(`#FBF9F6`) 바탕의 깔끔하고 신뢰감 있는 핀테크 UX.

---

## 2. 릴리즈 단계 및 로드맵

### Phase 1: MVP - 금융상품 비교 플랫폼 (2026년 4월 목표)

**목표**: 금감원 API 연동을 통한 예금/적금 금리 비교 서비스 런칭.

- **UI 구현**: 
    - 옅은 베이지 테마의 메인 대시보드 (`/`)
    - 상품 요약 카드 및 기간별 필터 시스템
    - 상품 상세 정보 페이지 (`/deposit/[id]`)
- **API 연동**: 
    - 금감원 금융상품 통합 비교 공시 API 연동
    - 서버 사이드 캐싱 전략 적용
- **Treia 이식**: 
    - 기존 Treia 랜딩 페이지를 `/treia` 경로로 이동 및 메뉴화

### Phase 2: 서비스 고도화 및 벤치마킹 (2026년 4월 ~ 5월)

**목표**: 토스/뱅크샐러드 수준의 금융 UX 및 Firebase 연동 기반 개인화.

- **금융 섹션 (Toss/BankSalad 벤치마킹)**:
    - **Toss 스타일**: 상품 핵심 혜택(예: "먼저 받는 이자")을 강조하는 카드형 UI 도입.
    - **BankSalad 스타일**: 우대금리 조건 충족 여부 체크리스트 및 만기 수령액 시뮬레이션 고도화.
    - **기능**: 공유하기(Web Share/카카오), 관심 상품 찜하기(Firebase) 구현.
- **인증 및 DB (Firebase)**:
    - **Auth**: Firebase 기반 소셜 로그인.
    - **Prefix 전략**: Treia와 데이터 혼선 방지를 위해 `fintable_` 프리픽스 사용 (예: `fintable_users`, `fintable_bookmarks`).

### Phase 3: 주거 서비스 고도화 (2026년 5월 ~ 6월)

**목표**: '내집다오' 수준의 임대주택 공고 및 청약 진단 기능 구축.

- **내집다오 벤치마킹**:
    - **청약 진단**: 간단한 문답을 통한 입주 자격(소득/자산) 자가 진단 기능.
    - **경쟁률 분석**: 과거 공고 데이터를 활용한 당첨 확률 시뮬레이션.
    - **공공주택 특화**: LH/SH 외 GH(경기), iH(인천) 등 지방공사 데이터 통합.
- **분양/임대 캘린더**: 월별 모집 일정을 한눈에 보는 캘린더 뷰.

## 3. 핵심 아키텍처 및 폴더 구조 계획

```text
app/
├── (fintable)/      ← 메인 금융 서비스 (Light 테마)
│   ├── page.tsx     ← 예금 리스트
│   └── [id]/        ← 상품 상세
├── treia/           ← 골드 자동매매 (Dark 테마 유지)
│   └── page.tsx
├── housing/         ← 임대주택 (3차 예정)
│   └── page.tsx
└── api/
    ├── deposits/    ← 금감원 API 프록시
    └── housing/     ← LH/SH API 연동
```
