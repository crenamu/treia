# 어드민 모니터링 개선 프로젝트

## 1. 현재 상황 분석
- **현상**: 어드민 페이지(`app/admin/page.tsx`)에서 EA의 모니터링 데이터가 실시간으로 반영되지 않거나 누락되는 느낌임.
- **구조**: 
    - EA가 `/api/license/update` POST 요청을 통해 상태를 업데이트함.
    - `route.ts`에서 Firebase Firestore의 `treia_licenses` 컬렉션에 데이터를 저장함.
    - 어드민 페이지에서 `getDocs`를 통해 데이터를 가져와 표시함.
- **추측되는 원인**: 
    - `lastUpdate` 판단 로직(70분 이내)이 너무 관대하거나, EA의 업데이트 주기와 맞지 않음.
    - 클라이언트 측에서 `fetchAll` 주기가 1분으로 설정되어 있어 체감상 늦을 수 있음.
    - 데이터 업데이트 시 `history` 배열에만 쌓이고 최상위 필드 반영에 문제가 있을 가능성.
    - 브라우저 탭 활성화 여부에 따른 갱신 중단 가능성.

## 2. 해결 계획 (설계도)
- **실시간성 강화**: 1분 주기 `setInterval` 대신 Firebase의 `onSnapshot`을 사용하여 데이터 변경 시 즉시 반영되도록 수정.
- **상태 판단 정교화**: `isOnline` 로직을 개선하여 더 정확한 온라인/오프라인 상태 표시.
- **데이터 로깅 강화**: `history` 데이터가 누락되지 않도록 서버 사이드 검증 로직 확인.
- **UI 개선**: 선택된 계좌의 상세 정보 표시 시 로딩 처리 및 데이터 가독성 향상.

## 3. 상세 사양 (시방서)
- **Firebase Real-time Update**: `app/admin/page.tsx`의 `MonitorTab` 컴포넌트에서 `onSnapshot` 적용.
- **온라인 상태 기준**: 마지막 업데이트 10분 이내일 경우만 온라인으로 표시.
- **에러 핸들링**: API 호출 실패 시 EA 로그와 서버 로그를 대조할 수 있도록 응답 메시지 세분화.
